## Flow 5: 商機 Pipeline (Deal Flow)

### 5A: 從情報建立商機

```mermaid
flowchart TD
    A["情報 Feed 中看到一則情報"] --> B["判斷：這有機會"]
    B --> C["Tap '建立商機'"]
    C --> D["商機建立卡片\n(預填客戶 + 痛點)"]

    D --> D1["確認/選擇客戶\n(從通訊錄或新建)"]
    D1 --> D2["商機名稱\n(AI 建議 or 手動)"]
    D2 --> D3["預估預算範圍\n[<100K] [100-500K] [500K-1M] [1M+] [未知]"]
    D3 --> D4["預估時程\n[本季] [下季] [半年內] [一年內] [未定]"]
    D4 --> E["✅ 商機已建立\n自動進入 L0 (潛在)"]

    E --> F["跳轉到商機詳情"]

    style E fill:#2d5a3d,stroke:#4ade80
```

### 5B: 商機 Pipeline 列表

兩種檢視模式，右上角切換按鈕：

```mermaid
flowchart TD
    A["打開 商機 Tab"] --> B{"檢視模式？\n(右上角切換)"}

    B -->|"緊急度排序"| V1["垂直列表\n依閒置天數排序\n(最需推進的在最上)"]
    B -->|"階段分組"| V2["垂直列表\n依階段分組(可摺疊)\nL0 / L1 / L2 / L3+"]

    V1 --> C["Tap 一張商機卡"]
    V2 --> C
    C --> D["商機詳情頁"]

    D --> D1["基本資訊\n客戶 · 痛點 · 預算 · 時程"]
    D --> D2["MEDDIC 進度\n(可展開填寫)"]
    D --> D3["文件追蹤\nNDA / MOU / 簡報"]
    D --> D4["搭配夥伴\n(已配對 + 建議)"]
    D --> D5["相關情報\n(自動關聯)"]
    D --> D6["下次行動\n(排會議/跟進)"]
    D --> D7["TBD 清單\n(待確認事項)"]
```

### 5C: 商機推進（狀態轉換）

```mermaid
flowchart TD
    A["商機詳情頁"] --> B["Tap '推進階段'"]
    B --> C{"MEDDIC\n關卡檢查"}
    C -->|"通過"| D["選擇新階段\nL0 → L1 → L2 → ..."]
    C -->|"未通過"| E["顯示缺少項目\n'需先完成：M, E'"]
    E --> F["填寫 MEDDIC\n(直接在此頁)"]
    F --> C

    D --> G["確認推進"]
    G --> H["自動提示：\n排下次會議？"]
    H -->|"排"| I["跳到行事曆\n新增會議"]
    H -->|"稍後"| J["回到商機詳情"]

    style D fill:#2d5a3d,stroke:#4ade80
    style E fill:#5a4a2d,stroke:#fbbf24
```

### 5D: 商機配對夥伴

```mermaid
flowchart TD
    A["商機詳情 → 夥伴區塊"] --> B{"已有配對夥伴？"}
    B -->|"有"| C["顯示已配對夥伴卡片\n能力 · 信任度 · 合作記錄"]
    B -->|"無"| D["系統建議\n(根據能力標籤 × 信任度)"]

    D --> E["顯示推薦夥伴\n排序 by 匹配度"]
    E --> F["Tap 選擇夥伴"]
    F --> G["加入此商機"]
    G --> C

    C --> H["手動搜尋\n+ 新增夥伴"]

    style D fill:#2d4a5a,stroke:#60a5fa
```

## Flow 6: 行事曆 (Calendar)

### 6A: 行事曆主視圖

```mermaid
flowchart TD
    A["打開 行事曆 Tab"] --> B["月視圖\n日期格有圓點標記"]

    B --> C["Tap 某一天"]
    C --> D["日視圖：\n該日所有事項"]

    D --> D1["🔵 會議\n中油 — 陳副廠長 14:00"]
    D --> D2["🟡 推進提醒\nA 食品 已 14 天未聯繫"]
    D --> D3["🔴 到期提醒\nB 科技 NDA 30天後到期"]

    D1 --> E["Tap 會議"]
    E --> F["會議詳情\n→ 會前準備包"]

    D2 --> G["Tap 推進提醒"]
    G --> H["跳到商機詳情"]

    D3 --> I["Tap 到期提醒"]
    I --> J["跳到文件追蹤"]
```

### 6B: 新增會議（從行事曆）

```mermaid
flowchart TD
    A["行事曆 → Tap + 或長按日期"] --> B["新增事項"]
    B --> C{"事項類型"}

    C -->|"會議"| D["選擇關聯商機\n(從商機列表)"]
    D --> D1["日期時間"]
    D1 --> D2["參與人\n(從通訊錄選)"]
    D2 --> D3["備註 (optional)"]
    D3 --> E["✅ 會議已排定"]

    C -->|"跟進提醒"| F["選擇關聯商機"]
    F --> F1["提醒日期"]
    F1 --> F2["提醒內容"]
    F2 --> G["✅ 提醒已設定"]

    style E fill:#2d5a3d,stroke:#4ade80
    style G fill:#2d5a3d,stroke:#4ade80
```

### 6C: 會前 → 開會 → 會後 完整循環

```mermaid
flowchart TD
    A["行事曆顯示\n今天有會議"] --> B["Tap 會議卡片"]
    B --> C["會前準備包"]

    C --> C1["📋 TBD 清單\n上次待確認的 3 件事"]
    C --> C2["📊 MEDDIC 進度\n目前 3/6"]
    C --> C3["📄 NDA: 已簽 ✓\nMOU: 待簽 ⚠️"]

    C3 --> D
    C4 --> D
    C5 --> D

    H --> I["提示：排下次會議？"]
```

## Flow 8: 文件上傳 + AI 解析

### 8A: 上傳文件（從商機詳情）

```mermaid
flowchart TD
    A["商機詳情 → 文件區塊"] --> B["Tap '+ 新增文件'"]
    B --> C{"上傳方式"}

    C -->|"貼上連結"| D["輸入 Google Drive URL"]
    D --> D1["API 抓取檔案\n(透過 Google Drive MCP)"]
    D1 --> D2{"抓取成功？"}
    D2 -->|"成功"| E["檔案已儲存\n顯示檔名 + 大小"]
    D2 -->|"失敗"| D3["錯誤提示：\n權限不足 / 連結無效"]
    D3 --> C

    C -->|"從裝置選擇"| F["系統檔案選擇器\n(PDF / PPT / PPTX / DOCX)"]
    F --> F1["上傳檔案"]
    F1 --> E

    E --> G{"檔案類型\n可 AI 解析？"}
    G -->|"是 (PPT/PPTX/PDF)"| H["自動 AI 解析"]
    G -->|"否 (其他)"| I["僅存檔\n標記為附件"]

    H --> J["解析結果卡片"]
    J --> J1["方案摘要\n(解決什麼問題)"]
    J --> J2["報價金額\n(如有)"]
    J --> J3["預期效益\n(ROI / 效率提升)"]
    J --> J4["關鍵數據\n(時程 / 規格 / 範圍)"]

    J --> K["確認 / 編輯解析結果"]
    K --> L["解析結果存入商機\n會前準備包也顯示"]

    style E fill:#2d5a3d,stroke:#4ade80
    style L fill:#2d5a3d,stroke:#4ade80
    style D3 fill:#5a2d2d,stroke:#f87171
```

### 8B: 文件類型與標籤

```mermaid
flowchart TD
    A["檔案上傳後"] --> B{"文件分類\n(自動 or 手動)"}

    B --> B1["簡報 (Proposal)\n方案 / 報價"]
    B --> B2["合約 (Contract)\nNDA / MOU"]
    B --> B3["附件 (Attachment)\n其他文件"]

    B1 --> C["AI 解析 + 顯示摘要"]
    B2 --> D["連結到 NDA/MOU 追蹤\n更新簽署狀態"]
    B3 --> E["僅存檔"]

    C --> F["商機詳情 · 文件區\n顯示 '已解析' 徽章"]
    D --> F
    E --> F

    style F fill:#2d5a3d,stroke:#4ade80
```

## Flow 10: 商機關閉（唯一路徑）

```mermaid
flowchart TD
    A["商機詳情頁"] --> B["Tap '關閉商機'"]
    B --> C{"確認原因"}

    C -->|"客戶明確拒絕"| D["選擇拒絕原因\n[預算不足] [時程不合]\n[選擇競品] [需求消失] [其他]"]
    C -->|"取消"| E["返回商機詳情"]

    D --> F["備註 (optional)"]
    F --> G["二次確認：\n確定關閉？"]
    G -->|"確定"| H["商機狀態 → 已關閉\n記錄關閉日期 + 原因"]
    G -->|"取消"| E

    H --> I["回到 Pipeline 列表\n(已關閉商機灰顯)"]

    style H fill:#3d3d3d,stroke:#9ca3af
```

## Flow 11: 情報關聯商機

```mermaid
flowchart TD
    A["新情報輸入完成\n(Flow 1)"] --> B{"是否關聯\n既有商機？"}

    B -->|"有相關商機"| C["AI 自動建議\n可能關聯的商機"]
    B -->|"無 / 不確定"| D["情報獨立存在\n(可稍後關聯)"]

    C --> E["顯示建議商機\n(1-3 個)"]
    E --> F{"使用者選擇"}
    F -->|"選擇一個"| G["情報關聯到該商機\n商機詳情自動更新"]
    F -->|"都不是"| D
    F -->|"建立新商機"| H["跳到 Flow 5A\n從情報建立商機"]

    D --> I["情報 Feed 中顯示\n(未關聯標記)"]
    I --> J["之後可手動關聯"]

    style G fill:#2d5a3d,stroke:#4ade80
```
