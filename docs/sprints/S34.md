## Stage 4: Retrospective & Refactor

### Fixes

- [x] `link_intel_to_deal` nested connection lock — moved `touch_deal` into same cursor

### Retro: What went well

- Clean break schema design — `nx_` prefix avoids all legacy conflicts
- Service pattern (raw SQL + psycopg2-style cursor) reused perfectly from S33
- Seed script caught the nested connection bug immediately
- All 14 tables + 9 services + 9 routers in one sprint

### Retro: What didn't go well

- SQLite single-writer lock caught us on nested `get_connection()` calls — must be careful with any service function that calls another service inside a `with get_connection()` block

### Retro: Action items for next sprint

- [ ] S35: Intel inbox frontend (capture → AI parse → card Q&A → save)
- [ ] Consider connection-passing pattern for service functions that compose other services

### Retro: AI collaboration notes

- Effective: writing all service files in parallel, seed script as integration test
- Effective: schema-first approach — designing all 14 tables upfront prevented rework

- Estimated: 3 hours
- Actual: ~1 hour
- Reason: no frontend work, pure backend CRUD is straightforward with established patterns
