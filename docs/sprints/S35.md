## Stage 2: Vibe Coding

### Implementation Log

1. **Layout overhaul**: Replaced legacy sidebar-only layout with mobile-first design
   - `bottom-nav.tsx`: 5-tab bottom nav with FAB center button (md:hidden)
   - `desktop-sidebar.tsx`: Desktop sidebar (hidden md:flex) with same navigation
   - `top-bar.tsx`: Page title + theme toggle
   - `theme-provider.tsx`: Dark/light toggle, persists to localStorage
2. **Nexus API client**: `lib/nexus-api.ts` — full TypeScript types + API methods for all 9 entity groups
3. **Intel capture flow**: `/capture` → text input → POST to API → redirect to Q&A
4. **Card Q&A flow**: `/capture/qa` — role question → branch (client: 5 questions, partner: 3 questions)
   - Single select: tap to advance
   - Multi-select: tap to select, confirm button
   - Skip: creates TBD via API, advances to next question
   - Progress bar + step counter
   - Finish: confirms intel, creates client/partner entity
5. **Deals pipeline**: `/deals` — urgency sort (default) / stage grouping toggle
   - Deal cards with idle days, stage badge, left border color by urgency
   - Collapsible stage sections
6. **Intel feed**: `/intel` — chronological list with status badges
7. **Contacts**: `/contacts` — sub-tabs for clients / partners, trust level badges
8. **Calendar**: `/calendar` — placeholder (S37)
9. **Home redirect**: `/` → `/deals`
10. Installed `lucide-react`, Inter font via Google Fonts CDN
