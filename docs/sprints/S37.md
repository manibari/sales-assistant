### Bug fix: Meeting prep data loading

**Problem**: `meetingId` was extracted from params but never used. The page called `meetingsByDate("")` then iterated all deals, breaking on the first one — always showing the first deal's data regardless of which meeting was selected.

**Fix**: Use `nxApi.calendar.getMeeting(meetingId)` to fetch the specific meeting, then load the associated deal by `meeting.deal_id`. Added `getMeeting` method to `nexus-api.ts`.
